<h2 id="serpent">Serpent</h2>
This is a simple deobfuscation challenge.

Once you replace a few variable names, you find that they will all run to form a single flag string, that is not being printed.
![](/ctfs/ctf@cit2025/serpent.webp)

So, we print out the flag, and we get:

`CIT{7777aKMpU9X3TqnD}`

<h2 id="babykeygen">Baby Keygen</h2>
1. The input length must be length 16
2. The first 4 characters must be `KEY_`
3. After we run the program 16 times, the program will print the flag
![](/ctfs/ctf@cit2025/babykeygen.webp)
`CIT{41jN8BKzz388}`
